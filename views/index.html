Node.js crash Course

What is Node.js?

Node.js is a JavaScript runtime built on Chrome's V8 JavaScript engine.

What is a JavaScript runtime?

A runtime is a program that runs a program written in a specific language. In this case, Node.js is a program that runs JavaScript code.

What is Chrome's V8 JavaScript Engine?

V8 is an open-source JavaScript engine developed by The Chromium Project for Google Chrome and Chromium web browsers. V8 compiles JavaScript code to native machine code instead of interpreting it or executing it as bytecode.

What can I do with Node.js?

Node.js is commonly used for backend services, APIs, and microservices. It is also used to build command-line tools and desktop applications.

What is npm?

npm is the package manager for the Node JavaScript platform. It puts modules in place so that node can find them, and manages dependency conflicts intelligently.

What is a package?

A package is a file or directory that is described by a package.json file. A package must contain a package.json file in order to be published to the npm registry.

What is a module?

A module is any file or directory in the node_modules directory that can be loaded by the Node.js require() function.

What is a dependency?

A dependency is a local package that a package.json file depends on.

What is a devDependency?

A devDependency is a package that is only necessary for local development and testing.

What is a peerDependency?

A peerDependency is a package that is only used when your package is installed as a dependency of another package.

What is a global dependency?

A global dependency is a package that is installed globally on your operating system.

What is a semantic version?

A semantic version is a version number that consists of three numbers separated by periods. The first number is the major version, the second number is the minor version, and the third number is the patch version.
